1.  Project Overview & Objectives
BiTranspiler is a bidirectional code transpilation tool that converts x86 Assembly (MASM syntax) to readable C++ and C++ back to x86 Assembly. Built entirely in C++, it demonstrates a thorough understanding of how low-level instructions map to high-level constructs — the central insight of the COAL course.

Objectives
Convert a defined subset of x86 MASM instructions into equivalent, commented C++ code
Convert core C++ constructs (variables, arithmetic, if/else, loops, functions) into MASM Assembly
Provide a clean terminal UI with a conversion summary, warnings, and formatted output files
Demonstrate COAL concepts — registers, stack, flags, control flow, and procedure calls — through implementation

3.  System Architecture
Both conversion directions follow the same three-stage pipeline:

Stage
ASM → C++ Pipeline
C++ → ASM Pipeline
1. Lexer
Tokenize .asm file — instructions, registers, labels, operands
Tokenize .cpp file — keywords, identifiers, operators, literals
2. Parser
Recognize multi-line patterns (CMP+JE = if, DEC+JNZ = loop, CALL/RET = function)
Parse expressions and control-flow structures into an IR tree
3. Code Gen
Emit formatted, commented C++ — registers mapped to variable names
Emit formatted MASM Assembly with labels, PROCs, and comments

A Warning Engine runs alongside both pipelines, logging any unrecognized patterns as comments in the output rather than failing. A main menu ties both pipelines together with file I/O and a conversion summary report.

7.  Real-World Applications
Education
Professors can use BiTranspiler to show C++ students the exact Assembly equivalent of their code — making low-level concepts immediately tangible
Assembly students can verify their logic by seeing the C++ translation of what they wrote
Instructors can generate exam questions: show one side, ask students to produce the other
Reverse Engineering & Security
Security analysts examining binaries can convert known Assembly patterns to C++ quickly, speeding up manual code review
Malware analysts can use the C++ output as a readable starting point for understanding Assembly routines
Embedded Systems & Legacy Code
Embedded developers can document Assembly-written routines in C++ for team members who are not Assembly-fluent
Legacy Assembly codebases can be partially transpiled to C++ to aid understanding before refactoring
Developer Tools
BiTranspiler demonstrates the same architecture as professional tools (Compiler Explorer, Ghidra, IDA Pro) at a teachable scale — making it a reference project for future Compiler Design coursework

8.  Conclusion
BiTranspiler is an original, technically ambitious project that goes beyond implementing Assembly , it requires understanding how both languages relate at a structural level, which is the deepest insight the COAL course develops. It has clear deliverables, a realistic 7-day timeline, genuine real-world value, and produces a compelling interactive demo. We respectfully submit this proposal for approval.
